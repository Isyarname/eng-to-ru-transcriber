# English-to-Russian Phonetic Transcriptor 🔤

Масштабируемый гибридный движок для автоматической транскрипции английского текста в кириллицу. Проект использует кастомный декларативный DSL для описания фонетических правил, автоматическую генерацию и кэширование регулярных выражений по требованию и двухуровневую систему поиска.

## ✨ Ключевые возможности
* **Универсальный DSL-движок**: Универсальный DSL-движок: Лингвистические правила полностью изолированы в виде декларативных конфигураций. Вы можете изменять, добавлять или полностью переписывать правила транслитерации, не меняя программный код самого компилятора и движка обработки.
* **Автоматическая компиляция и кэширование**: Скрипт автоматически собирает DSL-правила в оптимизированные пары регулярных выражений (`re.compile`) и сохраняет их в JSON-кэш при первом запуске (Lazy Initialization).
* **Гибридный поиск**: Модуль `eng_to_ipa_hybrid.py` совмещает мгновенный поиск по локальной базе данных (`en_US.txt`) с динамическим фолбэком на фонетическую библиотеку `eng_to_ipa`.
* **Токенизация**: Сохранение знаков препинания, переносов строк и оригинального регистра букв при переводе.

### 📐 Пайплайн обработки данных

Весь процесс работы программы делится на два независимых этапа: подготовку правил и обработку текста.

#### 1. Подготовка и кэширование правил
* **Проверка кэша**: Точка входа или внешний API-вызов обращаются к функции `convert()`, которая проверяет наличие файла `compiled_rules.json`.
* **Компиляция (`src/compiler.py`)**: Если файла нет, внутренний модуль считывает декларативные правила из `data/ipa_to_ru_rules.py`. Компилятор разворачивает лингвистические сокращения, автоматически генерирует правила для заглавных букв (учет регистра) и собирает регулярные выражения.
* **Сохранение**: Скомпилированные правила сериализуются в `compiled_rules.json`. При последующих запусках этот этап пропускается, обеспечивая старт с нулевой задержкой.

#### 2. Транскрипция текста (Вычислительный слой)
* **Интерфейс ввода-вывода (`main.py`)**: Точка входа для консольного запуска. Занимается исключительно чтением исходного `text.txt`, передачей строки в движок и записью результата в `output_text.txt`.
* **Оркестрация (`src/eng_to_ru.py`)**: Универсальная функция `convert()`. Если аргументы не переданы, она лениво парсит локальный словарь `data/en_US.txt` и подтягивает JSON-кэш правил, после чего координирует чистые вычисления.
* **Фонетический анализ (`src/eng_to_ipa_hybrid.py`)**: Реализует двухуровневый поиск. Если слово есть в словаре исключений — транскрипция берется оттуда, если нет — она генерируется «на лету» через библиотеку `eng_to_ipa`.
* **Транслитерация (`src/ipa_to_ru.py` + `src/transducer.py`)**: Полученный IPA-текст передается в модуль транслитерации. Чистый движок последовательно применяет скомпилированные регулярные выражения к строке, заменяя фонетические знаки на кириллицу.

## 📂 Структура проекта
```text
eng-to-ru-transcriptor/
│
├── main.py                  # Главная точка входа для запуска транскриптора
├── compiled_rules.json      # Автоматически генерируемый JSON-кэш скомпилированных правил
├── output_text.txt          # Файл с результатом работы программы (выходной текст)
├── requirements.txt
├── .gitignore
│
├── data/                    # Данные, словари и правила
│   ├── text.txt             # Исходный английский текст для транскрипции
│   ├── en_US.txt            # Локальный словарь для быстрого поиска и исключений
│   └── ipa_to_ru_rules.py   # Описание правил транслитерации на кастомном DSL
│
└── src/                     # Исходный код модулей движка
    ├── compiler.py          # Компилятор: парсит DSL-правила и собирает пары Regex
    ├── transducer.py        # Движок трансдьюсера: последовательное применение регулярных выражений
    ├── ipa_to_ru.py         # Модуль транслитерации IPA-символов в кириллицу
    ├── eng_to_ipa_hybrid.py # Гибридный модуль: интеграция с eng_to_ipa и словарем en_US.txt
    └── eng_to_ru.py         # Главный оркестратор: содержит универсальную функцию convert()
```
## 🚀 Установка и запуск

1. Клонируйте репозиторий:
```bash
git clone https://github.com
cd eng-to-ru-transcriptor
```

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

3. Поместите исходный английский текст в файл `data/text.txt`.

4. Запустите транскриптор:
```bash
python main.py
```

Результат работы будет автоматически сохранен в файл `output_text.txt` в корне проекта.
