# English-to-Russian Phonetic Transcriptor 🔤

Масштабируемый гибридный движок для автоматической транскрипции английского текста в кириллицу. Проект использует кастомный декларативный DSL для описания фонетических правил, автоматическую JIT-компиляцию в регулярные выражения и двухуровневую систему поиска.

## ✨ Ключевые возможности
* **Универсальный DSL-движок**: Лингвистическая логика транслитерации полностью отделена от программного кода. Правила описываются на лаконичном предметно-ориентированном языке (DSL).
* **Автоматическая компиляция и кэширование**: Скрипт автоматически собирает DSL-правила в оптимизированные пары регулярных выражений (`re.compile`) и сохраняет их в JSON-кэш при первом запуске (Lazy Initialization / Just-in-Time Caching).
* **Гибридный поиск**: Модуль `eng_to_ipa_hybrid.py` совмещает мгновенный поиск по локальной базе данных (`en_US.txt`) с динамическим фолбэком на фонетическую библиотеку `eng_to_ipa`.
* **Токенизация**: Сохранение знаков препинания, переносов строк и оригинального регистра букв при переводе.

## 📐 Архитектура и принцип работы

Проект реализован по принципу строгого разделения данных и логики управления:

[Входной текст data/text.txt] ➔ [Очистка/Токенизация]
│
▼
[Поиск в локальном en_US.txt] ──(Есть)──➔ [Вывод кириллицы]
│
(Нет)
▼
[eng_to_ipa_hybrid (IPA)] ➔ [Маппинг через compiled_rules.json] ➔ [Вывод]

### Пайплайн обработки правил:
1. **Описание правил (`data/ipa_to_ru_rules.py`)**: Правила контекстной замены букв и звуков задаются в декларативном виде на удобном DSL.
2. **Умное кэширование (`src/compiler.py`)**: При старте движок проверяет наличие скомпилированного файла `compiled_rules.json`. Если файла нет, компилятор разворачивает сокращения, преобразует правила в регулярные выражения и сохраняет кэш. Если файл есть — он мгновенно считывается из памяти, обеспечивая запуск с нулевой задержкой.
3. **Рендеринг (`src/transducer.py`)**: Модуль трансдьюсера последовательно и эффективно применяет скомпилированные регулярные выражения к тексту.

## 📂 Структура проекта
```text
eng-to-ru-transcriptor/
│
├── main.py                  # Главная точка входа для запуска транскриптора
├── compiled_rules.json      # Автоматически генерируемый JSON-кэш скомпилированных правил
├── output_text.txt          # Файл с результатом работы программы (выходной текст)
├── requirements.txt         # Файл с внешними зависимостями проекта
├── .gitignore               # Исключение кэша и локальных результатов из Git
│
├── data/                    # Данные, словари и правила
│   ├── text.txt             # Исходный английский текст для транскрипции
│   ├── словарь en_US.txt     # Локальный словарь для быстрого поиска и исключений
│   └── ipa_to_ru_rules.py   # Описание правил транслитерации на кастомном DSL
│
└── src/                     # Исходный код модулей движка
    ├── compiler.py          # Компилятор: парсит DSL-правила и собирает пары Regex
    ├── transducer.py        # Движок трансдьюсера: последовательное применение регулярных выражений
    ├── ipa_to_ru.py         # Модуль транслитерации IPA-символов в кириллицу
    ├── eng_to_ipa_hybrid.py # Гибридный модуль: интеграция с eng_to_ipa и словарем en_US.txt
    └── eng_to_ru.py         # Главный оркестратор: координирует работу всех слоев системы
```
## 🚀 Установка и запуск

1. Клонируйте репозиторий:
```bash
git clone https://github.com
cd eng-to-ru-transcriptor
```

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

3. Поместите исходный английский текст в файл `data/text.txt`.

4. Запустите транскриптор:
```bash
python main.py
```

Результат работы будет автоматически сохранен в файл `output_text.txt` в корне проекта.
